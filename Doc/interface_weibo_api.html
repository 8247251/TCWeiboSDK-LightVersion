详细描述

定义了腾讯微博授权，api请求接口

注解
典型调用顺序 1. 初始化 ——> 通过webview或者SSO授权WeiboApi对象 ——>请求接口
典型调用顺序 2. 初始化 ——> 使用已有授权初始化WeiboApi对象 ——>请求接口
授权优先级：SSO可用时，优先使用SSO进行授权
重刷授权： 当使用SSO授权时，不提供重刷授权机制

处理微博客户端唤回后的回调数据

注解

返回

- (int) requestWithParams: (NSMutableDictionary *)  params
apiName: (NSString *)  apiName
httpMethod: (NSString *)  httpMethod
delegate: (id)  delegate 

发送api请求, 腾讯微博OPenApi请求都可以通过这个接口进行访问，如何调用请参考官网文档

注解
授权应用信息使用当前weiboapi对象中的数据
需实现WeiboRequestDelegate 中的didReceiveRawData 和 didFailWithError
参数
INPUTparams 请求参数对， key 为NSString类型的参数名， value 一般情况下为NSString 类型对象。发表图片时，value必须为NSData，或者UIImage对象。
INPUTapiName api名称，例如收听某人的url是https://open.t.qq.com/api/friends/add， 它的apiname就是 friends/add
INPUThttpMethod GET or POST
INPUTdelegate 回调delegate 应实现WeiboRequestDelegate
返回
返回请求序号,序号为递增
